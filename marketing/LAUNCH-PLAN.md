# ForgeX Launch Plan

Roadmap authority note:
- `docs/VERSIONS.md` is the canonical source for what the current version means.
- Launch copy must not imply a stronger maturity level than the active version gate allows.

`npx forgex -> permanence -> XRPL`

- GitHub
- X
- Discord
- Instagram
- Beacons

- Kick

## Channel Intent

### GitHub

- make the repo feel complete on first visit
- README must convert cold traffic
- release copy must be short, premium, and usable immediately
- release copy must stay inside the active version gate

### X

- focus on screenshot-worthy proof
- use short lines and strong visuals
- avoid long thread energy unless metrics justify it later

### Discord

- treat as warm audience, not cold-market advertising
- keep the tone direct and useful
- ask for screenshots and first-run reactions

### Instagram

- use the artist-driven visual identity and polished UI
- carousel angle: premium product surface, not tutorial content

### Beacons

- refresh the link hub on Day 0 so all traffic paths are current
- link priority:
  - GitHub repo
  - README and install path
  - X
  - Discord

## Launch Message

ForgeX is the obvious way to deploy Solidity contracts to XRPL when you want the experience to feel immediate, premium, and final.

## Success Markers

- people post screenshots of the permanence block
- the README converts cold visitors into `npx forgex`
- X launch post drives stars and tries
- Discord audience actually runs it
- GitHub release feels like a product drop, not a dev dump
